# 数据同步运维手册

> 从远程 DataAPI 拉取数据到本地 `data/dameng_mirror.db`（SQLite）。

## 工具

| 脚本 | 用途 | 调度方式 |
|------|------|----------|
| `ops/scripts/sync/sync_mirror.py` | 月度数据同步（37 个 SYNC_TABLE） | 手动 / cron |
| `ops/scripts/sync/auto_sync.py` | 事件驱动同步（检测上游版本变更） | cron（建议每小时） |
| `ops/scripts/sync/sync_daily.py` | 日粒度营收+车流同步 | cron（每日凌晨） |

**数据库**: `data/dameng_mirror.db`  
**远程 API**: `http://111.229.213.193:18071/api/dynamic`  
**云平台直连**: `https://api.eshangtech.com`（日营收/日车流专用）

## 常用命令

```bash
# 诊断：查看哪些月份缺数据
python ops/scripts/sync/sync_mirror.py

# 拉指定月份（增量，已有的自动跳过）
python ops/scripts/sync/sync_mirror.py --month 202603

# 清理旧数据后重拉（数据有误时用）
python ops/scripts/sync/sync_mirror.py --month 202603 --clean

# 只拉指定接口
python ops/scripts/sync/sync_mirror.py --endpoint customer_age_ratio --month 202603

# 全量刷新 B/D 类全量表
python ops/scripts/sync/sync_mirror.py --full-refresh

# 自动模式（跳过确认，给 AI agent 用）
python ops/scripts/sync/sync_mirror.py --month 202603 --yes

# 只跑验证不拉数据
python ops/scripts/sync/sync_mirror.py --verify-only
```

### auto_sync（事件驱动）

```bash
# 检查哪些表需要同步（只看不做）
python ops/scripts/sync/auto_sync.py --dry-run

# 执行同步（自动检测版本差异，只拉有变更的表）
python ops/scripts/sync/auto_sync.py

# 查看同步日志
python ops/scripts/sync/auto_sync.py --log
```

### sync_daily（日粒度）

```bash
# 拉昨天数据
python ops/scripts/sync/sync_daily.py

# 拉指定日期
python ops/scripts/sync/sync_daily.py --date 2026-03-14

# 回填最近 7 天
python ops/scripts/sync/sync_daily.py --backfill 7

# 只拉营收 / 只拉车流
python ops/scripts/sync/sync_daily.py --only revenue
python ops/scripts/sync/sync_daily.py --only traffic
```

## 7 类同步策略

| 类型 | 含义 | 拉法 | 表数 | 示例接口 |
|------|------|------|------|----------|
| **A** | 省级月度 | 按月 GET，一次拉全省 | 16 | revenue, traffic, finance |
| **A_CHILDREN** | A 类子表展开 | A 类拉完后展开 children | 1 | vehicle_ownership |
| **B** | 全量快照 | 不带月份，一次 GET | 11 | basic_info, merchant_shops, revenue_ranking |
| **C** | 服务区×月 | 遍历 150 SA × 逐月 | 5 | customer_age_ratio, contract_merchant |
| **D** | 服务区全量 | 遍历 150 SA × 1 次 | 3 | merchant_profit_loss, project_warning |
| **E** | 枚举×月 | 遍历枚举值 × 逐月 | 1 | traffic_warning (4 种类型) |
| **F** | 日粒度 | auto_sync 专用，按日拉 | 1 | daily_closing |

> SYNC_TABLE 共 37 项（A×16 + A_CHILDREN×1 + B×11 + C×5 + D×3 + E×1），EXTRA_SYNC_TABLE 1 项（F类 daily_closing），合计 **38 个表配置**  
> C 类最耗时（150 SA × 月数 × 接口数），全量同步约 30-60 分钟

## 同步流程

```
1. 诊断模式查缺失
   python ops/scripts/sync_mirror.py
   → 输出每张表的行数、最新月份、缺失月份

2. 确认后拉取
   python ops/scripts/sync_mirror.py --month YYYYMM --yes

3. 验证
   python ops/scripts/sync_mirror.py --verify-only
   → 五维度验证：表存在/行数/月份覆盖/SA覆盖/内容抽查
```

## 日粒度同步（sync_daily.py）

独立于月度同步体系，由 `sync_daily.py` 管理。数据源为云平台直连 API。

| 本地表 | 数据源 API | 粒度 | 保留天数 |
|--------|-----------|------|----------|
| LOCAL_DAILY_REVENUE | GetRevenueReportByDate | 1 服务区 × 1 天 | 90 天 |
| LOCAL_DAILY_TRAFFIC | GetSECTIONFLOWList | 1 服务区 × 1 天 | 90 天 |
| NEWSECTIONFLOWLIST | NEWGetSECTIONFLOWList | 1 服务区 × 1 天 | 90 天 |

**预算衍生指标**（存入 DB，LLM 直接引用无需计算）：
- 营收：`MOBILE_RATE`（移动支付占比）、`AVG_TICKET`（客单价）、`EAST_RATE`/`WEST_RATE`（东西区占比）
- 车流：`ENTRY_RATE`（入区率）、`ENTRY_RATE_EAST`/`ENTRY_RATE_WEST`（东西区入区率）

**工具对接**：`get_daily_revenue` 和 `get_daily_traffic` 已改为 **local-first** 模式，先查本地表，无数据时 fallback 远端 API。

## 事件驱动同步（auto_sync.py）

基于上游 `sync-state` API 的版本对比机制，只同步有变更的表。

| 组件 | 说明 |
|------|------|
| `_SYNC_STATE` 表 | 记录每张表的本地版本号（`synced_version`） |
| `_SYNC_LOG` 表 | 记录每次同步的状态、行数变化、错误信息 |
| 版本对比 | `local.synced_version < upstream.version` 时触发同步 |
| 联动更新 | `revenue_report_v2` 更新后自动触发 `NEWGETREVENUEREPORT_SHOPS` 展平 |

## 已知问题

| 问题 | 说明 | 处置 |
|------|------|------|
| customer_profile 2026年只有 51 个 SA | 202512 有 122 个，202601 骤降到 51 个 | 重拉 `--endpoint customer_age_ratio --month 202601 --clean` |
| B/D 类增量模式不重拉 | 已有数据时跳过 | 用 `--full-refresh` 强制重拉 |
| C 类耗时长 | 134 SA × 月数次 API 调用 | 可 `--endpoint xxx` 只拉单表 |
| ~~ALL_MONTHS 包含 202601~~ | `pull_api_data.py` 曾把 202601 纳入月份范围，导致 cloud_revenue_* 表存了 13 个月累计 | **已修复**(2026-03-11): `ALL_MONTHS` 改为 `range(1,13)` 即 202501-202512；3 个 ORM 表已从 `dameng_mirror.db` 重新聚合 |

## 故障排查

| 现象 | 可能原因 | 解法 |
|------|---------|------|
| API 调用超时 | 远程 API 不可达 | `curl http://111.229.213.193:18071/api/dynamic/GetServerpartList/` 测通 |
| 表不存在 | 首次拉取 | 脚本会自动建表 |
| 数据行数为 0 | API 返回空 | 检查 `Result_Data.List` 是否有数据 |
| 月份覆盖不全 | 增量跳过已有 | `--clean --month YYYYMM` 清理后重拉 |
| SA 覆盖率低 | C 类某些 SA 无数据 | 正常现象，部分 SA 在源 API 无数据 |
| 展平表缺某月份 | 源表（如 NEWGETSHOPSABFILIST）该月未同步，展平时自动跳过 | `--month YYYYMM --endpoint sabfi` 补同步源表，展平自动执行 |
